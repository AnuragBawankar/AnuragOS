## Completed

- Added command-line argument support using `sys.argv`
- Added `help` command
- Introduced centralized logging (`logger.py`)
- Refactored `Orchestrator` to use logging
- Refactored `WorkspaceAgent` to use logging
- Refactored `MacTools` to use logging
- Tested end-to-end workspace startup successfully

## Architecture Decisions

- Logging is centralized in a single module.
- Business logic no longer uses `print()` directly.
- Tool-specific logging resides in `MacTools`.
- Command input is now provided by the terminal instead of being hardcoded.

## Lessons Learned

- Keep responsibilities separated.
- Refactor incrementally and test after every change.
- Small, focused commits are easier to review and maintain.
